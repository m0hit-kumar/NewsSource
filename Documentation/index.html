NewsSource API Documentation

Introduction

NewsSource is a web-based API that provides users with trending news on any given topic. It collects data from various sources such as Google Trends, Wikipedia, Twitter, and news articles. The data is then summarized and analyzed for confidence using machine learning models, specifically Decision Tree Classifier, Gradient Boosting Classifier, and Logistic Regression. This documentation aims to provide a guide on how to use the NewsSource API to fetch data on trending topics and how to interpret the data.

Getting Started

To use the API, you dont need need an API key. Just go through the the documenatations and you will learn various endpoints that are available to use.

API Endpoints

The NewsSource API provides the following endpoints:

  • - / - This endpoint gives you idea of how to can use the API and it has the documentation for the API
    Example:
    https://newssource.pythonanywhere.com
  • - /query/query_string - This endpoint allows users to search for news on a specific topic. The parameter is the topic that the user wants to search for. The API then returns the trending news, tweets, and Wikipedia summary related to the topic.
    Example:
    https://newssource.pythonanywhere.com/query/bitcoin
  • - /trendingNews - This endpoint provides users with trending news on various topics. The API returns the latest news articles sorted by their publication date.
    Example:
    https://newssource.pythonanywhere.com/trendingNews
  • - /api - This endpoint returns a demo data file in JSON format to help users understand the data structure returned by the API.
    Example:
    https://newssource.pythonanywhere.com/api

Response Formats

The API returns JSON data by default.

  • _id: A unique identifier for the event.
  • title: The title of the event, in this case, "Attack of 26/11".
  • description: A description of the event, including details about the attacks and the organization responsible for them.
  • imageUrl: A URL pointing to an image related to the event, in this case, a picture of the aftermath of the attacks.
  • dataSources: A comma-separated list of sources used to gather information about the event, in this case, Wikipedia and The Hindu.
  • sourceConfidence: A confidence score (out of 100) indicating the reliability of the sources used to gather information about the event.
  • category: A category for the event, in this case, "Incident".
  • dateCreated: The date and time when the event was created in ISO 8601 format.
  • __v: A version field used by the database, which can be ignored for most purposes.

Authentication and Authorization

This API does not require authentication or authorization for accessing any of the endpoints. However, if you plan on using this API for a production environment, it is recommended to add proper authentication and authorization mechanisms.

Rate Limiting

The API does not currently implement any rate limiting. However, if the API is used extensively, it may cause performance issues, and it is recommended to implement rate limiting to prevent abuse.

Future Improvements

  • - Adding support for pagination to prevent overwhelming the client with large amounts of data.
  • - Adding support for sorting and filtering to allow the client to customize the response data.
  • - Improving the accuracy of the machine learning model used to summarize and classify news articles.
  • - Adding more sources of data for the API to fetch information from.
